# 04 — Wings

Wings is the game server daemon that manages Docker containers for each game server and communicates with the Panel.

> ⚠️ Wings cannot be tested on Windows. Run this on the Raspberry Pi directly.

## Prerequisites

- Raspberry Pi 5 running Raspberry Pi OS Bookworm 64-bit
- Docker installed (guaranteed if the Panel is already running)
- Pelican Panel reachable

---

## Installation

There are two ways to run Wings. **Docker is recommended.**

### Option A — Docker (Recommended)

Uses the official Wings container image. Easier to update and manage.

1. Copy the example env file and adjust if needed:

```bash
cp wings/.env.example wings/.env
```

2. Start Wings:

```bash
cd wings
docker compose up -d
```

Wings is now running as a container and will restart automatically on reboot.

---

### Option B — Native (Binary)

Runs Wings as a systemd service directly on the host. Use this if you cannot run Docker-in-Docker or prefer a bare-metal setup.

Run the setup script as root:

```bash
sudo bash wings/setup.sh
```

The script will:
- Verify Docker is installed
- Create required directories (`/etc/pelican`, `/var/run/wings`)
- Download the Wings binary for ARM64
- Install and register the systemd service

---

## Connecting Wings to the Panel

This step is the same for both installation methods. The config contains a token generated by the Panel.

1. Open the Panel → Admin → Nodes → Create New
2. Fill in the node details (name, IP of the Pi, port 8080)
3. Click the node → tab **Configuration**
4. Copy the config block

Now save the config on the Pi:

```bash
sudo nano /etc/pelican/config.yml
# paste the config, save with CTRL+X
```

> Alternatively use the **Auto Deploy Command** button in the Panel — it does this step automatically.

---

## Testing

Start Wings in debug mode to verify the connection:

**Docker:**
```bash
cd wings
docker compose logs -f
```

**Native:**
```bash
sudo wings --debug
```

Check for errors. If Wings connects to the Panel successfully, the node status in the Panel will turn green.

---

## Starting / Stopping

### Docker

```bash
# Start
docker compose up -d

# Stop
docker compose down

# Restart
docker compose restart wings
```

### Native (systemd)

```bash
# Enable and start on boot
sudo systemctl enable --now wings

# Status
sudo systemctl status wings

# Stop / Restart
sudo systemctl stop wings
sudo systemctl restart wings
```

## File Locations

| Path | Description |
|---|---|
| `/etc/pelican/config.yml` | Wings configuration (from Panel) |
| `/var/lib/pelican/volumes/` | Game server files |
| `/tmp/pelican/` | Temporary install files |
| `/usr/local/bin/wings` | Wings binary (native only) |
| `/var/run/wings/daemon.pid` | PID file (native only) |
